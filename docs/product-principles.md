# Capture Inbox Product Principles

## Purpose

Capture Inbox exists to help people make better decisions about where to
invest their limited time and attention.

It is **not** a bookmarking application. It is **not** a read-it-later
application. It is an **Attention Operating System**.

Every feature should move the product closer to helping users answer one
question:

> **"Is this worth my time?"**

------------------------------------------------------------------------

# Core Principles

## 1. Attention is More Valuable than Information

The internet has an unlimited supply of information.

A user's attention is limited.

Every feature should optimize for protecting attention rather than
increasing content consumption.

------------------------------------------------------------------------

## 2. Help Users Decide, Not Just Consume

The application should answer:

-   Should I read this?
-   Should I watch this?
-   Should I postpone it?
-   Should I ignore it?

Decision-making comes before summaries.

------------------------------------------------------------------------

## 3. AI Should Reduce Cognitive Load

AI exists to remove unnecessary work.

Users should spend less time:

-   deciding
-   searching
-   organizing
-   comparing

and more time learning.

------------------------------------------------------------------------

## 4. Structure Over Raw Text

Whenever possible, convert content into structured knowledge.

Examples include:

-   Topics
-   Difficulty
-   Practicality
-   Estimated effort
-   Target audience
-   Learning outcomes
-   Prerequisites

Structured information enables better search, recommendations and future
features.

------------------------------------------------------------------------

## 5. Explain Every Recommendation

AI should never behave like a black box.

Every recommendation should include a reason.

Examples:

-   Contains practical examples.
-   Similar to resources you've already completed.
-   Mostly historical background.
-   Requires knowledge you haven't learned yet.

Users should understand *why* the recommendation was made.

------------------------------------------------------------------------

## 6. Personalization is Earned

Recommendations should become better by observing:

-   learning goals
-   interests
-   completed content
-   ignored content
-   available time

Personalization should improve gradually instead of relying on
assumptions.

------------------------------------------------------------------------

## 7. Every Capture Should Become More Valuable Over Time

A saved URL should evolve into knowledge.

Capture → Understanding → Evaluation → Recommendation → Learning →
Connected Knowledge

The value of a capture should increase as the system learns more about
it and about the user.

------------------------------------------------------------------------

## 8. AI Assists, Humans Decide

The application recommends.

The user decides.

Recommendations should be clear, transparent and easy to override.

------------------------------------------------------------------------

## 9. Optimize for Learning ROI

The product should maximize:

## Learning gained

Time invested

Success means helping users learn more in less time---not consuming more
content.

------------------------------------------------------------------------

## 10. Build for Long-Term Thinking

Features should compose into a larger system.

Every capability should strengthen future capabilities.

For example:

-   Structured metadata enables personalization.
-   Personalization enables prioritization.
-   Prioritization enables daily planning.
-   Daily planning enables long-term learning.

------------------------------------------------------------------------

# Product Test

Before building a feature, ask:

1.  Does this save the user time?
2.  Does it reduce decision fatigue?
3.  Does it improve learning?
4.  Does it help answer "Is this worth my attention?"
5.  Will it still make sense as the product grows?

If the answer to most of these questions is "no", reconsider the
feature.

------------------------------------------------------------------------

# Product Motto

> **Capture less. Learn more. Spend your attention wisely.**
